Open\+SSL is a robust, commercial-\/grade, full-\/featured Open Source Toolkit for the TLS (formerly SSL), DTLS and QUIC (currently client side only) protocols.

The protocol implementations are based on a full-\/strength general purpose cryptographic library, which can also be used stand-\/alone. Also included is a cryptographic module validated to conform with FIPS standards.

Open\+SSL is descended from the SSLeay library developed by Eric A. Young and Tim J. Hudson.

The Open\+SSL toolkit includes\+:


\begin{DoxyItemize}
\item {\bfseries{libssl}} an implementation of all TLS protocol versions up to TLSv1.\+3 (\href{https://tools.ietf.org/html/rfc8446>}{\texttt{ RFC 8446}}), DTLS protocol versions up to DTLSv1.\+2 (\href{https://tools.ietf.org/html/rfc6347>}{\texttt{ RFC 6347}}) and the QUIC (currently client side only) version 1 protocol (\href{https://tools.ietf.org/html/rfc9000>}{\texttt{ RFC 9000}}).
\item {\bfseries{libcrypto}} a full-\/strength general purpose cryptographic library. It constitutes the basis of the TLS implementation, but can also be used independently.
\item {\bfseries{openssl}} the Open\+SSL command line tool, a swiss army knife for cryptographic tasks, testing and analyzing. It can be used for
\begin{DoxyItemize}
\item creation of key parameters
\item creation of X.\+509 certificates, CSRs and CRLs
\item calculation of message digests
\item encryption and decryption
\item SSL/\+TLS/\+DTLS and client and server tests
\item QUIC client tests
\item handling of S/\+MIME signed or encrypted mail
\item and more...
\end{DoxyItemize}
\end{DoxyItemize}\hypertarget{md__c_1_2_users_2namph_2_downloads_2openssl_2openssl-3_82_81_2_r_e_a_d_m_e_autotoc_md953}{}\doxysection{\texorpdfstring{Download}{Download}}\label{md__c_1_2_users_2namph_2_downloads_2openssl_2openssl-3_82_81_2_r_e_a_d_m_e_autotoc_md953}

Source code tarballs of the official releases can be downloaded from \href{https://www.openssl.org/source}{\texttt{ www.\+openssl.\+org/source}}. The Open\+SSL project does not distribute the toolkit in binary form.

However, for a large variety of operating systems precompiled versions of the Open\+SSL toolkit are available. In particular, on Linux and other Unix operating systems, it is normally recommended to link against the precompiled shared libraries provided by the distributor or vendor.

We also maintain a list of third parties that produce Open\+SSL binaries for various Operating Systems (including Windows) on the \href{https://wiki.openssl.org/index.php/Binaries>}{\texttt{ Binaries}} page on our wiki.\hypertarget{md__c_1_2_users_2namph_2_downloads_2openssl_2openssl-3_82_81_2_r_e_a_d_m_e_autotoc_md955}{}\doxysubsection{\texorpdfstring{For Testing and Development}{For Testing and Development}}\label{md__c_1_2_users_2namph_2_downloads_2openssl_2openssl-3_82_81_2_r_e_a_d_m_e_autotoc_md955}
Although testing and development could in theory also be done using the source tarballs, having a local copy of the git repository with the entire project history gives you much more insight into the code base.

The official Open\+SSL Git Repository is located at \href{https://git.openssl.org>}{\texttt{ git.\+openssl.\+org}}. There is a Git\+Hub mirror of the repository at \href{https://github.com/openssl/openssl>}{\texttt{ github.\+com/openssl/openssl}}, which is updated automatically from the former on every commit.

A local copy of the Git Repository can be obtained by cloning it from the original Open\+SSL repository using \begin{DoxyVerb}git clone git://git.openssl.org/openssl.git
\end{DoxyVerb}
 or from the Git\+Hub mirror using \begin{DoxyVerb}git clone https://github.com/openssl/openssl.git
\end{DoxyVerb}
 If you intend to contribute to Open\+SSL, either to fix bugs or contribute new features, you need to fork the Open\+SSL repository openssl/openssl on Git\+Hub and clone your public fork instead. \begin{DoxyVerb}git clone https://github.com/yourname/openssl.git
\end{DoxyVerb}
 This is necessary because all development of Open\+SSL nowadays is done via Git\+Hub pull requests. For more details, see Contributing.\hypertarget{md__c_1_2_users_2namph_2_downloads_2openssl_2openssl-3_82_81_2_r_e_a_d_m_e_autotoc_md956}{}\doxysection{\texorpdfstring{Build and Install}{Build and Install}}\label{md__c_1_2_users_2namph_2_downloads_2openssl_2openssl-3_82_81_2_r_e_a_d_m_e_autotoc_md956}
