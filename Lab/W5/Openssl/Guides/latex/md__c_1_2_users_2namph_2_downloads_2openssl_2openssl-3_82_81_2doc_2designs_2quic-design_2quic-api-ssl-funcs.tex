\chapter{Behaviour of SSL functions on QUIC SSL objects}
\hypertarget{md__c_1_2_users_2namph_2_downloads_2openssl_2openssl-3_82_81_2doc_2designs_2quic-design_2quic-api-ssl-funcs}{}\label{md__c_1_2_users_2namph_2_downloads_2openssl_2openssl-3_82_81_2doc_2designs_2quic-design_2quic-api-ssl-funcs}\index{Behaviour of SSL functions on QUIC SSL objects@{Behaviour of SSL functions on QUIC SSL objects}}
This document is a companion to the \doxysectlink{md__c_1_2_users_2namph_2_downloads_2openssl_2openssl-3_82_81_2doc_2designs_2quic-design_2quic-api}{QUIC API Overview}{0} which lists all SSL functions and controls and notes their behaviour with QUIC SSL objects.

The Category column is as follows\+:


\begin{DoxyItemize}
\item {\bfseries{Global}}\+: These API items do not relate to SSL objects. They may be stateless or may relate only to global state.

Can also be used for APIs implemented only in terms of other public libssl APIs.
\item {\bfseries{Object}}\+: Object management APIs. Some of these may require QUIC-\/specific implementation.
\item {\bfseries{HL}}\+: Handshake layer API.

These calls should generally be dispatched to the handshake layer, unless they are not applicable to QUIC. Modifications inside the handshake layer for the QUIC case may or may not be required.
\item {\bfseries{CSSM}}\+: Connection/\+Stream State Machine. API related to lifecycle of a connection or stream. Needs QUIC-\/specific implementation.
\item {\bfseries{ADP}}\+: App Data Path. Application-\/side data path API. QUIC-\/specific implementation.
\item {\bfseries{NDP}}\+: Net Data Path. Network-\/side data path control API. Also includes I/O ticking and timeout handling.
\item {\bfseries{RL}}\+: Record layer related API. If these API items only relate to the TLS record layer, they must be disabled for QUIC; if they are also relevant to the QUIC record layer, they will require QUIC-\/specific implementation.
\item {\bfseries{Async}}\+: Relates to the async functionality.
\item {\bfseries{0-\/RTT}}\+: Relates to early data/0-\/\+RTT functionality.
\item {\bfseries{Special}}\+: Other calls which defy classification.
\end{DoxyItemize}

The Semantics column is as follows\+:


\begin{DoxyItemize}
\item {\bfseries{🟩U}}\+: Unchanged. The semantics of the API are not changed for QUIC.
\item {\bfseries{🟧C}}\+: Changed. The semantics of the API are changed for QUIC.
\item {\bfseries{🟦N}}\+: New. The API is new for QUIC.
\item {\bfseries{🟥\+TBD}}\+: Yet to be determined if semantic changes will be required.
\end{DoxyItemize}

The Applicability column is as follows\+:


\begin{DoxyItemize}
\item {\bfseries{🟦U}}\+: Unrelated. Not applicable to QUIC — fully unrelated (e.\+g. functions for other SSL methods).
\item {\bfseries{🟥\+FC}}\+: Not applicable to QUIC (or not currently supported) — fail closed.
\item {\bfseries{🟧\+NO}}\+: Not applicable to QUIC (nor not currently supported) — no-\/op.
\item {\bfseries{🟩A}}\+: Applicable.
\end{DoxyItemize}

The Implementation Requirements column is as follows\+:


\begin{DoxyItemize}
\item {\bfseries{🟩\+NC}}\+: No changes are expected to be needed (where marked \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}, dispatch to handshake layer).

{\bfseries{Note}}\+: Where this value is used with an applicability of {\bfseries{FC}} or {\bfseries{NO}}, this means that the desired behaviour is already an emergent consequence of the existing code.
\item {\bfseries{🟨C}}\+: Modifications are expected to be needed (where marked \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}, dispatch to handshake layer with changes inside the handshake layer).
\item {\bfseries{🟧\+QSI}}\+: QUIC specific implementation.
\item {\bfseries{🟥\+QSA}}\+: QUIC specific API.
\end{DoxyItemize}

The Status column is as follows\+:


\begin{DoxyItemize}
\item {\bfseries{🔴\+Pending Triage}}\+: Have not determined the classification of this API item yet.
\item {\bfseries{🟠\+Design TBD}}\+: It has not yet been determined how this API item will work for QUIC.
\item {\bfseries{🟡\+TODO}}\+: It has been determined how this API item should work for QUIC but it has not yet been implemented.
\item {\bfseries{🟢\+Done}}\+: No further work is anticipated to be needed for this API item.
\end{DoxyItemize}

Notes\+:


\begin{DoxyItemize}
\item †1\+: Must restrict which ciphers can be used with QUIC; otherwise, no changes.
\item †2\+: ALPN usage must be mandated; otherwise, no changes.
\item †3\+: NPN usage should be forced off as it should never be used with QUIC; otherwise, no changes.
\item †4\+: Controls needing changes are listed separately.
\item †5\+: TLS compression and renegotiation must not be used with QUIC, but these features are already forbidden in TLS 1.\+3, which is a requirement for QUIC, thus no changes should be needed.
\item †6\+: Callback specified is called for handshake layer messages (TLSv1.\+3).
\item †7\+: Tickets are issued using {\ttfamily NEW\+\_\+\+TOKEN} frames in QUIC and this will require handshake layer changes. However these APIs as such do not require changes.
\item †8\+: Use of post-\/handshake authentication is prohibited by QUIC.
\item †9\+: QUIC always uses AES-\/128-\/\+GCM initially. We need to determine when and what ciphers we report as being in use.
\item †10\+: Not supporting async for now.
\item †11\+: Since these functions only configure cipher suite lists used for TLSv1.\+2, which is never used for QUIC, they do not require changes, and we can allow applications to configure these lists freely, as they will be ignored.
\end{DoxyItemize}

QUIC always uses AES-\/128-\/\+GCM for Initial packets. At this time the handshake layer has not negotiated a ciphersuite so it has no “current” cipher. We could return AES-\/128-\/\+GCM here, but it seems reasonable to just return NULL as the encryption is mostly for protection against accidental modification and not “real” encryption. From the perspective of the Handshake layer encryption is not active yet. An application using QUIC can always interpret NULL as meaning AES-\/128-\/\+GCM is being used if needed as this is implied by using QUIC.

A. We return NULL here, because it allows applications to detect if a ciphersuite has been negotiated and NULL can be used to infer that Initial encryption is still being used. This also minimises the changes needed to the implementation.\hypertarget{md__c_1_2_users_2namph_2_downloads_2openssl_2openssl-3_82_81_2doc_2designs_2quic-design_2quic-api-ssl-funcs_autotoc_md298}{}\doxysubsection{\texorpdfstring{What should {\ttfamily SSL\+\_\+\+CTX\+\_\+set\+\_\+cipher\+\_\+list} do?}{What should {\ttfamily SSL\+\_\+\+CTX\+\_\+set\+\_\+cipher\+\_\+list} do?}}\label{md__c_1_2_users_2namph_2_downloads_2openssl_2openssl-3_82_81_2doc_2designs_2quic-design_2quic-api-ssl-funcs_autotoc_md298}
