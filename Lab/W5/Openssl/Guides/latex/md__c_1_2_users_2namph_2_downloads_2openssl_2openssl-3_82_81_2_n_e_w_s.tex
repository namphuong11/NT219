Open\+SSL 3.\+2.\+0 is a feature release adding significant new functionality to Open\+SSL.

This release incorporates the following potentially significant or incompatible changes\+:


\begin{DoxyItemize}
\item The default SSL/\+TLS security level has been changed from 1 to 2.
\item The {\ttfamily x509}, {\ttfamily ca}, and {\ttfamily req} apps now always produce X.\+509v3 certificates.
\item Subject or issuer names in X.\+509 objects are now displayed as UTF-\/8 strings by default.
\end{DoxyItemize}

This release adds the following new features\+:


\begin{DoxyItemize}
\item Support for client side QUIC, including support for multiple streams (RFC 9000)
\item Support for Ed25519ctx, Ed25519ph and Ed448ph in addition to existing support for Ed25519 and Ed448 (RFC 8032)
\item Support for deterministic ECDSA signatures (RFC 6979)
\item Support for AES-\/\+GCM-\/\+SIV, a nonce-\/misuse-\/resistant AEAD (RFC 8452)
\item Support for the Argon2 KDF, along with supporting thread pool functionality (RFC 9106)
\item Support for Hybrid Public Key Encryption (HPKE) (RFC 9180)
\item Support for SM4-\/\+XTS
\item Support for Brainpool curves in TLS 1.\+3
\item Support for TLS Raw Public Keys (RFC 7250)
\item Support for TCP Fast Open on Linux, mac\+OS and Free\+BSD, where enabled and supported (RFC 7413)
\item Support for TLS certificate compression, including library support for zlib, Brotli and zstd (RFC 8879)
\item Support for provider-\/based pluggable signature algorithms in TLS 1.\+3 with supporting CMS and X.\+509 functionality

With a suitable provider this enables the use of post-\/quantum/quantum-\/safe cryptography.
\item Support for using the Windows system certificate store as a source of trusted root certificates

This is not yet enabled by default and must be activated using an environment variable. This is likely to become enabled by default in a future feature release.
\item Support for using the IANA standard names in TLS ciphersuite configuration
\item Multiple new features and improvements to CMP protocol support
\end{DoxyItemize}

The following known issues are present in this release and will be rectified in a future release\+:


\begin{DoxyItemize}
\item Provider-\/based signature algorithms cannot be configured using the Signature\+Algorithms configuration file parameter (\#22761)
\end{DoxyItemize}

This release incorporates the following documentation enhancements\+:


\begin{DoxyItemize}
\item Added multiple tutorials on the Open\+SSL library and in particular on writing various clients (using TLS and QUIC protocols) with libssl

See \href{https://www.openssl.org/docs/manmaster/man7/ossl-guide-introduction.html}{\texttt{ Open\+SSL Guide}}.
\end{DoxyItemize}
